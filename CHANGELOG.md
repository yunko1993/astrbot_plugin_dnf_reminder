# Changelog

## 1.6.6 - 2026-06-16

- 控制台提醒存在时，调度和 `/提醒测试` 只使用控制台提醒，不再合并旧 `reminders.json` 提醒。
- 控制台提醒支持为纯群号配置平台前缀，默认 `aiocqhttp`，自动拼成合法群会话 ID。
- 控制台提醒不再读取旧版全局群目标和 @ 所有人配置，避免升级后重复或误 @。

## 1.6.5 - 2026-06-16

- `/提醒测试` 改为测试当前所有启用提醒，包括控制台配置的提醒。
- 合并调度时会跳过行为完全相同的重复提醒，减少旧命令提醒和控制台提醒同时存在时的重复发送。

## 1.6.4 - 2026-06-16

- 控制台提醒新增标题字段，折叠列表优先显示标题，便于区分多条提醒。
- 提醒列表命令会显示配置中的标题。

## 1.6.3 - 2026-06-16

- 将控制台配置改为单个“控制台提醒列表”，每条提醒可单独配置时间、内容、发送目标和 @ 对象。
- 支持控制台提醒选择“不 @ / @ 指定人 / @ 所有人”。
- 命令添加的提醒和控制台配置的提醒分开管理、合并调度，避免互相覆盖。
- 隐藏保留旧版配置项，尽量兼容已存在的群目标和发送设置。

## 1.6.2 - 2026-06-16

- 将插件配置改为分组展示：发送设置、群目标设置、@ 提醒设置。
- 群目标配置改为列表控件，便于在 AstrBot 控制台添加多个群。
- 兼容读取旧版本的平铺配置，升级后无需手动迁移配置。
- 修正 `metadata.yaml` 和 README 中的 GitHub 仓库地址。

## 1.6.1 - 2026-06-16

- 添加提醒时支持直接识别群聊中的真实 `@成员`。
- 保留手写 `@QQ号` 的兼容写法。
- 保存提醒内容时会清理开头的 CQ at 标记或 `@QQ号` 前缀。

## 1.6.0 - 2026-06-16

- 添加提醒时支持可选 `@QQ号`。
- 群消息发送时优先生成真实 At 组件。
- 提醒列表展示已配置的 @ 目标。

## 1.5.1 - 2026-03-30

- 优化定时任务调度，避免 scheduler 尚未就绪导致提醒未注册。
- 定时触发时将发送任务调度回主事件循环。
- 增加旧数据目录迁移和兼容读取。

## 1.5.0 - 2026-03-30

- 支持发送到配置群。
- 支持群消息 @ 所有人。
- 新增插件配置 schema。
